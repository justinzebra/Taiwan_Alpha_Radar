# technical_eod_v1 調參分析與 v2 候選方向

本文記錄使用既有歷史預測結果，尋找「較準訊號共同條件」的初步分析。

> 結論先講：可以用過往答案調整策略，但不能直接把歷史最佳條件硬套成正式模型。這次分析顯示，最穩定的共同因子不是單一個股分數，而是「市場廣度」加上「強勢但未過熱」。

## 1. 分析目的

問題：

> 目前 `technical_eod_v1` 能否透過過往每日對答案，找出更準的條件，修正推薦策略？

本次分析目標：

- 找出命中率與超額報酬較好的共同條件。
- 避免單純過度擬合。
- 區分「可考慮做 v2」與「只是近期行情剛好有效」。

## 2. 分析資料

資料來源：本機 `backend/taiwan_alpha_radar.db`

樣本：

| 項目 | 數值 |
|---|---:|
| 預測期間 | 2025-11-11 至 2026-06-17 |
| 預測筆數 | 4,787 |
| 股票池 | 40 檔 |

```text
訓練期：2025-11-11 至 2026-04-06
驗證期：2026-04-07 至 2026-06-17
```

這不是最終嚴格樣本外驗證，只是第一輪 rule discovery。

## 4. 單因子結果

### 4.1 市場廣度是最穩定共同因子

規則：

```text
signal_score >= 60
market breadth >= 0.5
```

`market breadth` 定義：

```text
當日股票池中，上漲股票數 / 股票池股票數
```

結果：

| 區間 | Horizon | 超額報酬 | 方向命中 |
|---|---:|---:|---:|
| 訓練期 | 1 日 | +0.15% | 45.1% |
| 訓練期 | 3 日 | +0.48% | 48.6% |
| 訓練期 | 5 日 | +0.53% | 51.3% |
| 驗證期 | 1 日 | +0.38% | 57.5% |
| 驗證期 | 3 日 | +1.03% | 61.9% |
| 驗證期 | 5 日 | +0.90% | 60.3% |

解讀：

- 這是目前最值得採用的條件。
- 當市場不是多數股票上漲時，技術動能排名容易失效。
- 也就是說：個股技術強，最好要有市場廣度配合。

### 4.2 RSI 50 至 70 適合中短期，不一定提升隔日

規則：

```text
signal_score >= 60
50 <= RSI <= 70
```

結果：

| 區間 | Horizon | 超額報酬 | 方向命中 |
|---|---:|---:|---:|
| 訓練期 | 1 日 | +0.03% | 46.5% |
| 訓練期 | 3 日 | +0.55% | 51.5% |
| 訓練期 | 5 日 | +0.87% | 52.9% |
| 驗證期 | 1 日 | +0.19% | 54.5% |
| 驗證期 | 3 日 | +0.35% | 55.4% |
| 驗證期 | 5 日 | +0.39% | 57.5% |

解讀：

- RSI 50 至 70 確實比極端 RSI 更健康。
- 但它不是隔日命中率的主因。
- 比較適合當作「避免過熱」或「中短期品質」條件。

### 4.3 成交量確認不能單獨使用

規則：

```text
signal_score >= 60
volume_confirm = true
```

結果：

| 區間 | Horizon | 超額報酬 | 方向命中 |
|---|---:|---:|---:|
| 訓練期 | 1 日 | +0.06% | 47.4% |
| 訓練期 | 3 日 | -0.04% | 51.4% |
| 訓練期 | 5 日 | -0.11% | 49.3% |
| 驗證期 | 1 日 | -0.04% | 47.5% |
| 驗證期 | 3 日 | +0.28% | 54.0% |
| 驗證期 | 5 日 | +0.25% | 61.3% |

解讀：

- 帶量上漲本身不保證隔日繼續漲。
- 它比較適合搭配市場廣度與 RSI 使用。

### 4.4 族群差異

驗證期 `signal_score >= 60` 的 1 日結果：

| 族群 | 超額報酬 | 方向命中 |
|---|---:|---:|
| 半導體 | +0.50% | 56.9% |
| AI | +0.22% | 54.9% |
| 電動車 | +0.15% | 54.8% |
| 光通訊 | +0.06% | 53.7% |
| 金融 | -0.06% | 60.2% |
| 機器人 | -0.14% | 55.9% |
| 傳產 | -0.43% | 45.2% |
| 散熱 | -0.43% | 49.6% |

解讀：

- 近期半導體、AI、電動車較適合技術動能。
- 金融命中率高但超額報酬不佳，可能是波動較低。
- 傳產、散熱在驗證期對目前技術規則較不友善。
- 族群效果可能高度受市場 regime 影響，不能永久化。

## 5. 多因子候選

### 5.1 最強近期組合

規則：

```text
50 <= RSI <= 70
momentum_20d >= 5
ma_state = bull_stack
volume_confirm = true
```

驗證期 1 日：

| 指標 | 數值 |
|---|---:|
| 超額報酬 | +0.99% |
| 方向命中 | 57.9% |
| 勝率 | 57.9% |
| 覆蓋天數 | 25 |
| 平均每日檔數 | 2.7 |

訓練期：

| 指標 | 數值 |
|---|---:|
| 超額報酬 | +0.33% |
| 方向命中 | 50.0% |

解讀：

- 很像「精選強勢股」條件。
- 但每日檔數偏少，不適合取代綜合排名。
- 可作為高品質標籤，不應直接變成唯一推薦策略。

### 5.2 較平衡候選

規則：

```text
signal_score >= 60
50 <= RSI <= 70
market breadth >= 0.5
volume_confirm = true
```

驗證期 1 日：

| 指標 | 數值 |
|---|---:|
| 超額報酬 | +0.87% |
| 方向命中 | 60.6% |
| 勝率 | 60.6% |
| 覆蓋天數 | 28 |
| 平均每日檔數 | 4.3 |

訓練期：

| 指標 | 數值 |
|---|---:|
| 超額報酬 | +0.01% |
| 方向命中 | 43.7% |

解讀：

- 近期效果很好。
- 但訓練期命中率差，可能是近期行情貼合。
- 可以列為 v2 觀察條件，但不能直接宣稱已穩定有效。

### 5.3 最穩健候選

規則：

```text
signal_score >= 60
market breadth >= 0.5
```

解讀：

- 雖然沒有最強組合漂亮，但跨訓練期與驗證期都改善超額報酬。
- 最適合先加入 v2 作為市場狀態過濾。
- 若市場廣度低於 50%，應降低推薦強度或標成「觀察」。

## 6. 建議策略修正方向

不建議直接修改 `technical_eod_v1`，因為它已經是基準模型，應保留可比性。

建議新增：

```text
technical_eod_v2_candidate
```

或在 UI 上先加入：

```text
高品質訊號標籤
```

### 6.1 v2 候選規則

#### 市場狀態過濾

```text
if market_breadth < 0.5:
    降低推薦強度
```

建議不要完全排除，因為弱勢市場也可能有反彈股；但應明確標為低品質環境。

#### 高品質訊號

```text
signal_score >= 60
market_breadth >= 0.5
50 <= RSI <= 70
volume_confirm = true
```

用途：

- 作為每日 Top 10 裡的品質標籤。
- 作為更嚴格的候選清單。
- 不取代原本排名。

#### 觀察週期調整

目前結果顯示：

```text
3 至 5 個交易日 > 隔日
```

因此頁面與文案應避免強調「明天一定準」，改成：

> 收盤後建立未來 3 至 5 個交易日觀察清單。

## 7. 不建議採用的調整

### 7.1 只提高分數門檻

例如：

```text
score >= 75
score >= 80
```

目前沒有證據顯示分數越高就一定越準。

近期某些極高分股票有效，但這可能只是行情偏強造成。

### 7.2 單獨使用成交量

`volume_confirm` 單獨使用沒有穩定提升隔日效果。

### 7.3 依單一族群固定加權

半導體、AI 近期較好，但族群輪動快，不應永久固定加分。

比較好的方式是：

- 顯示族群 tab。
- 追蹤各族群近期命中率。
- 讓族群強弱成為動態資訊，而不是硬編碼加分。

## 8. 下一步建議

### 短期

1. 保留 `technical_eod_v1`。
2. 加上「市場廣度」欄位與標籤。
3. 加上「高品質訊號」標籤。
4. 將主要觀察週期改成 3 至 5 日。

### 中期

1. 新增 `technical_eod_v2_candidate`。
2. 和 v1 同時產生預測。
3. 分開保存 outcome。
4. 至少累積 1 至 2 個月新資料後再決定是否升級為正式 v2。

### 長期

1. 加入交易成本。
2. 改用除權息還原價格。
3. 改用隔日開盤進場績效作為主要可交易指標。
4. 使用 rolling walk-forward 驗證，而不是一次切分。

## 9. 總結

目前找到的最大公因數：

```text
市場廣度 >= 50%
+ 個股技術分數 >= 60
+ RSI 位於 50 至 70
+ 有量價確認
```

但真正最穩的第一條是：

```text
市場廣度 >= 50%
```

因此策略修正不應只是改個股分數，而應加入「市場狀態」判斷。

建議採用的產品表述：

> `technical_eod_v1` 是基準技術排序；v2 候選應先加入市場廣度過濾與高品質訊號標籤，再透過未來樣本驗證，不直接用歷史最佳組合取代現有模型。
